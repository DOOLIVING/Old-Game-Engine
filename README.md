### 📜 Прототип системы Blueprint (Визуальное скриптирование)

Добавлен прототип **визуального редактора логики** (Blueprint System) — важный шаг к созданию игр без написания кода вручную.

* **Визуальные скрипты**: Теперь вы можете создавать простую логику поведения объектов, соединяя ноды в графе. Например: «При столкновении → Проиграть звук → Уничтожить объект».
* **События и действия**: Реализованы базовые узлы: `OnCollision`, `OnKeyPressed`, `SpawnObject`, `SetMaterial`, `PlaySound`, `PrintToConsole`.
* **Live-редактирование**: Изменения в Blueprint применяются в запущенной сцене без перекомпиляции (для простых скриптов).
* **Интеграция с Web-редактором**: Визуальный редактор доступен в веб-версии (Beta, функционал ~40%).
* **В разработке**: Переменные, ветвления (If/Else), циклы, пользовательские функции и экспорт Blueprint в JSON.

### 🚀 Что это даёт?
- Геймдизайнеры могут прототипировать механику за минуты.
- Художники по окружению — добавлять интерактивные элементы (движущиеся платформы, вращающиеся двери).
- Программисты — создавать кастомные ноды через простой API.

### 🔧 Технические детали
- Система построена на **графе выполнения** с поддержкой приоритетов узлов.
- Ноды компилируются во внутренний байт-код для производительности.
- Полная совместимость с физическим движком (AABB, Raycasting) и шейдерной архитектурой.

### 📦 Другие улучшения в 3.2.1
- **Стабильность**: Исправлены утечки памяти при многократной перезагрузке сцен в Web-версии.
- **UI**: Добавлена панель свойств для Blueprint-нод.
